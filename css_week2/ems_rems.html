Rems vs Ems

Ems

Ems are a multiplication of the measurement by the inherited font-size. Elements will inherit properties, such as font-size, from parent elements if no property is set for that specific element. This paragraph has a size of 20px, because the html element has a font-size of 20px. There is no CSS rule for paragraphs defined in this style sheet.

If you create an h2 element and apply a font-size: 2em; CSS property, then the h2 element will be 2em = 2 * 20px = 40px.

I am 40px at 100% screensize

One problem with ems, is that they compound. This is demonstrated by the 2 lists on the previous page. Here's another example of this:

Outer inner outer

Why is this happening? The span element has a font-size of 2em. An nested span element will use the multiplier on its parent (the first span element in this example).

So, the first span element (containing the words 'outer outer') will have a computed font size of:

2 * 20px (html font size) = 40px.

The second span will have a computed font size of:

2 * 40px (the computed size of the first span) = 80px.

The rem unit was created to deal with this problem.

Rems

The rem measurement works on the root of the html page, the html element. This eliminates the compounding problem because the the calculation is always based on the font-size property of the html element.
